# Flutter 项目结构说明

## 一句话：文件很多，但你平时只改很少一部分

- **你要写的代码** 几乎都在 `lib/` 里（现在只有 `main.dart`）。
- 其他大量文件是：**多平台支持**、**构建/工具生成**、**IDE 配置**，多数**不用改**。

---

## 整体是怎么运作的

```
┌─────────────────────────────────────────────────────────────┐
│  lib/main.dart  ← 你的 Flutter/Dart 业务代码（核心）          │
│  pubspec.yaml   ← 依赖、资源、项目名（必看）                  │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│  Flutter 引擎：把 Dart 代码编译成各平台可执行程序              │
└─────────────────────────────────────────────────────────────┘
         │           │           │           │           │
         ▼           ▼           ▼           ▼           ▼
    android/     ios/       web/      windows/    macos/  linux/
    (安卓)      (苹果)     (网页)     (桌面)      (桌面)  (桌面)
```

- 你主要在 **`lib/`** 里写 UI 和逻辑。
- **android / ios / web / windows / macos / linux** 是各平台的“壳子”，负责启动 Flutter、权限、图标等，一般只在改应用名、图标、权限时才会动。

---

## 按「要不要管」分类

### 1. 你需要关心的（很少）

| 路径 | 作用 |
|------|------|
| **lib/** | 所有 Dart 业务代码。新建页面、逻辑都放这里。 |
| **pubspec.yaml** | 项目名、依赖包、资源/字体声明。加新包或图片要改这里。 |
| **analysis_options.yaml** | 静态检查/代码规范（可选调）。 |
| **test/** | 单元/组件测试。 |
| **README.md** | 项目说明（可选）。 |

### 2. 平台壳子（偶尔改）

| 路径 | 作用 | 何时动 |
|------|------|--------|
| **android/** | 安卓工程（Gradle、MainActivity、权限）。 | 改应用名、包名、图标、权限。 |
| **ios/** | 苹果工程（Xcode、Info.plist）。 | 改应用名、图标、权限、描述。 |
| **web/** | 网页入口（index.html、manifest）。 | 改网页标题、PWA 配置。 |
| **windows/** | Windows 桌面壳子。 | 改应用名、图标（少见）。 |
| **macos/** | macOS 桌面壳子。 | 同上。 |
| **linux/** | Linux 桌面壳子。 | 同上。 |

### 3. 自动生成 / 工具（不用改，很多在 .gitignore）

| 路径或类型 | 作用 |
|------------|------|
| **build/** | 编译产物，可删，下次运行会重新生成。 |
| **.dart_tool/** | Dart/Flutter 工具缓存。 |
| **.idea/** | IDE 配置（运行配置等）。 |
| **\*GeneratedPluginRegistrant\*** | 插件注册代码，Flutter 按依赖自动生成。 |
| **ios/Flutter/ephemeral/** | iOS 构建时的临时配置。 |
| **macos/Flutter/ephemeral/** | macOS 同上。 |
| **pubspec.lock** | 依赖版本锁定，一般提交到 Git，不手改。 |
| **.metadata** | Flutter 创建项目时的元信息。 |

---

## 那么多文件都有用吗？

- **对「能跑起来」来说**：有用。每个平台都要自己的配置和入口，所以文件多。
- **对「你日常开发」来说**：  
  - **有用且要管的**：主要是 `lib/`、`pubspec.yaml`，偶尔 `android/`/`ios/` 等改名字、图标。  
  - **有用但不用改**：各平台目录里的默认配置、以及生成的 `Generated*`、`ephemeral` 等。  
  - **可删可忽略**：`build/`、`.dart_tool/`（删了会再生成）；`.idea/` 看团队是否要提交。

总结：**文件多是因为要支持 6 个平台 + 构建/工具生成**，但**你 99% 的时间只动 `lib/` 和 `pubspec.yaml`**，其它有用但多数不用改。
