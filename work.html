5. 后端接口关系

功能页面接口/地址结果
Cloud 上传 Maipdf2026Screen POST https://maipdf.com/pdf/r2upload.php 返回 path / file_id。App 登录后会先同步 PHP session，并带 Cookie 请求；当前上传接口本身主要保存文件，不单独写用户表。
Cloud 生成链接 Maipdf2026Screen POST https://maipdf.com/pdf/maipdf2026_backend.php 返回 HTML，App 解析出链接、Read Code、Modify Code。带 PHP session cookie 后，后端现有 $dengru 会识别为登录邮箱，并在现有记录逻辑里把用户名和文件路径关联起来。
Cloud 登录同步 Maipdf2026Screen / 首页登录 POST https://maipdf.com/6/firebase-register.php + POST https://maipdf.com/6/firebase-session-login.php MaiPdfCloudAuthService 用 Firebase 的 email 和 uid 同步 Cloud PHP 登录，并保存 PHPSESSID / dc cookie 给 Cloud 上传和生成链接使用。
DRM 打包 DrmSecureSharePage POST https://drm.maipdf.com/api/app/licenses/create App 只发送文件元数据、content_sha256、server_half、次数和过期；后端返回 license_id / mod_code。
DRM 阅读解锁 DrmProtectedViewerPage 内 locked HTML POST https://drm.maipdf.com/api/unlock 由 locked HTML 内部 JS 调用，用于扣次数、校验过期、获取解密 key half。
DRM 授权管理 DrmLicenseManagePage POST https://drm.maipdf.com/api/licenses/:lid/check|extend|revoke|delete App 原生页面调用网页端同一套管理 API；登录用户走 Bearer token，匿名/手动管理走 modification code。
DRM 我的授权列表 DrmLicenseManagePage GET https://drm.maipdf.com/api/me/licenses / POST /api/me/bind 登录后列出账号拥有的 license；也可以把旧的匿名 license 通过 license_id + mod_code 绑定进当前账号。

6. AustraliaGuide 里的 TS/JS 文件关系

后端/模板文件谁调用它当前状态作用

functions/api/unlock.ts .maipdf 内 locked HTML 的 JavaScript 调用 正在用 阅读时校验授权、扣次数，并返回混入 decoy 的 server_half 候选 parts。

functions/api/licenses/[lid]/delete.ts drm_license_manage_page.dart 通过 HTTPS POST 调用 正在用 App 管理页软删除授权，数据库记录保留，但读者会被阻止打开。
functions/api/pack.ts 网页端 drm.maipdf.com 网页端保留 现有网页端 pack，不影响 App 本地 pack。
functions/api/app/pack.ts 目前 Flutter App 不再调用 备用 之前新增的 App 专用后端 pack 版本；现在主线改为本地 pack，可保留作兼容/回退。
functions/lib/appViewerTemplate.ts functions/api/app/pack.ts 备用 后端 pack 备用模板，复用网页 viewer 但移除 Adsense。当前本地 pack 使用 Dart 内置轻量 HTML 模板。

这个文件是「在线 MaiPDF 云分享」这一整页的原生 Flutter 实现，对应网页版 maipdf2026.html。用户在这里：① 上传 PDF → ② 设置阅读次数/会话/水印等 → ③ 生成链接和二维码。下面按「和 PHP 的对应关系」+ 代码结构逐段说明，方便从 PHP 转过来的同学理解。

📦 知识点：Flutter 里「一个页面」是什么
在 Flutter 里没有「HTML 文件」的概念，一个「页面」就是一个 Widget（通常是一个 StatefulWidget）。Maipdf2026Screen 就是一个 Widget，负责整页的 UI 和逻辑，相当于 PHP 里「输出整页 HTML + 处理表单」的那一整套。

1. 文件开头的 import

import 'dart:async';      // TimeoutException 等
import 'dart:convert';    // jsonDecode
import 'package:flutter/material.dart';  // 界面组件
import 'package:flutter/services.dart'; // Clipboard
import 'package:http/http.dart' as http; // 发 HTTP 请求，类似 PHP 的 curl/file_get_contents
import 'package:file_picker/file_picker.dart'; // 选文件
import 'package:qr_flutter/qr_flutter.dart';   // 画二维码
import 'package:url_launcher/url_launcher.dart'; // 打开外部链接

和 PHP 的 require / include 类似，这里把要用到的库「引进来」。http 用来请求你的 r2upload.php 和 maipdf2026_backend.php。

2. 两个类：StatefulWidget + State

class Maipdf2026Screen extends StatefulWidget { ... }
class _Maipdf2026ScreenState extends State<Maipdf2026Screen> { ... }

Flutter 里「有状态」的页面要拆成两个类：

  • StatefulWidget：只负责「描述这个页面是谁」，本身几乎不写逻辑，只写 createState() => _Maipdf2026ScreenState()。
  • State：真正的逻辑和状态（变量、网络请求、表单值）都写在 _Maipdf2026ScreenState 里。名字前的下划线 _ 表示「只在当前文件内可见」，类似 PHP 的 private。
📦 知识点：为什么分 Widget 和 State
Flutter 为了性能会频繁「重建」Widget；如果状态和 Widget 写在一起，重建时状态可能被清空。所以把「会变化的数据」放在 State 里，这样重建界面时状态还在，只是根据状态重新画一遍 UI。

Widget 与 State 关系示意：

上图：StatefulWidget 只负责通过 createState() 创建对应的 State；State 持有所有状态变量，并提供 build() 方法返回当前界面。调用 setState() 后 Flutter 会重新执行 build() 更新界面。

_currentStep当前步骤 1/2/3，控制显示「上传 / 配置 / 结果」哪一块
_uploadStatus上传区显示的文字（如 "Ready"、"正在上传…"、"上传失败"）
_filePath上传成功后后端返回的 path，即 PHP 里发给 backend 的 sender
_fileId后端返回的 file_id，提交表单时会一起 POST 过去
_uploading / _submitting是否正在上传 / 正在提交，用来显示转圈、禁用按钮
_telegramChatIdControllerTelegram 读提醒的 chat_id（对应后端 mailalert，仅数字）；开启「Read Alerts」时必填才有提醒
_limitController 等输入框的「控制器」：既存当前输入值，又绑定到输入框显示（见下）
_resultLink, _readCode, _modifyCode第三步显示的链接、读码、改码，从后端返回的 HTML 里解析出来
📦 知识点：TextEditingController
PHP 里表单是 <input name="limit" value="...">，服务端用 $_POST['limit'] 读。Flutter 里没有服务端，输入框的值要自己「拿着」：TextEditingController 就是「拿着」这段文字的对象，controller: _limitController 绑到 TextField 上，用户输入会写进 controller，代码里用 _limitController.text 读、用 _limitController.text = "..." 写。

4. 生命周期：initState 和 dispose

@override
void initState() {
  super.initState();
  _loadBootstrap();  // 页面一打开就请求 bootstrap 接口
}

@override
void dispose() {
  _limitController.dispose();
  // ... 其他 controller
  super.dispose();    // 页面关闭时释放 controller，避免内存泄漏
}

initState：页面第一次出现时执行一次，类似 PHP 里「页面加载时先发一个 AJAX 拉配置」。
dispose：页面被关闭时执行，要把自己 new 出来的 Controller 都 dispose() 掉。

5. 和服务器打交道的几个方法

  • _loadBootstrap()：GET maipdf2026_backend.php?action=bootstrap，拿 JSON，把网络状态之类写到 _uploadStatus，用于页面顶部提示。
  • _pickAndUpload()：用 FilePicker 选一个 PDF → 用 http.MultipartRequest POST 到 r2upload.php（和网页里用 FormData 上传一样）→ 解析返回的 JSON 拿 path / file_id → 存到 _filePath、_fileId，并 setState 切到 Step 2。
  • _submitForm()：把 Step 2 的表单（limit、password、darkmode、zhangai、expiration_ts 等）用 MultipartRequest POST 到 maipdf2026_backend.php，和网页里点「Create」一样；后端返回整页 HTML，再用 _parseResultFromHtml(html) 从 HTML 里抠出链接和码。
📦 知识点：async / await
Dart 里发 HTTP 请求是「异步」的，不会卡住界面。写法：Future<void> _pickAndUpload() async { ... await request.send(); ... }。async 表示函数内部可以用 await；await 表示「等这个网络请求结束再往下执行」，类似 JS 的 async/await 或 PHP 里「等 curl 返回再处理」。
📦 知识点：setState
只要改了 State 里的变量（例如 _uploadStatus = "上传完成"），必须包在 setState(() { ... }) 里，Flutter 才会重新执行 build 把界面更新出来。不调 setState 只改变量，界面不会变。

6. 从后端 HTML 里解析结果：_parseResultFromHtml

后端返回的是整页 HTML，不是 JSON。所以用正则从 HTML 里找：

  • 带 id="myInput" 的 input 的 value="..." → 完整链接（注意 PHP 输出时 value 在 id 前面，所以正则先匹配 value="([^"]*)"[^>]*id="myInput"）。
  • result-message / result-password 里若有 result-chip 再取读码/改码。

解析出的链接存到 _resultLink 和 _resultLinkController.text，第三步就显示这个链接和二维码。

7. 界面是怎么「画」出来的：build

Widget build(BuildContext context) 是 Flutter 里「描述当前界面长什么样」的方法，每次 setState 后会重新执行。返回的是一个「组件树」：

Scaffold(                    // 整页骨架：AppBar + 内容区
  appBar: AppBar(...),
  body: BackgroundDecoration(
    child: ListView(
      children: [
        _buildStepProgress(),   // 顶部 1/2/3 步骤条
        if (_currentStep >= 1) _buildSection1(),  // 上传区
        if (_currentStep >= 2) _buildSection2(),  // 配置表单
        if (_currentStep >= 3) _buildSection3(),  // 链接 + 二维码
      ],
    ),
  ),
)

根据 _currentStep 决定显示哪几块，和 PHP 里 if ($step >= 2) { echo "..." ; } 逻辑一样。

📦 知识点：Widget 树
Flutter 界面是由一堆「Widget」嵌套出来的树：Scaffold → ListView → _SectionCard → TextField / FilledButton 等。每个 Widget 负责一块区域，有的管布局（如 Row、Column），有的管显示（如 Text、TextField）。没有 HTML/CSS，布局用 Row、Column、padding、SizedBox 等组合出来。
